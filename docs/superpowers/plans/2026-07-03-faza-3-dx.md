### Task 2: #10 — dry-run dla narzędzi mutujących

**Files:**
- Modify: `src/tools/mod.ts`, `src/tools/script-create.ts`, `src/tools/prefab.ts`, `src/tools/config-create.ts`, `src/tools/layout-create.ts`, `src/tools/project.ts`
- Test: `tests/tools/dry-run.test.ts` (nowy, zbiorczy) lub rozszerzenia istniejących testów per narzędzie

**Interfaces:**
- Każde narzędzie tworzące pliki dostaje `dryRun: z.boolean().default(false).describe("Preview what would be created/written without touching disk")`.
- Gdy `dryRun === true`: narzędzie zwraca odpowiedź opisującą planowane pliki (ścieżki + treść lub podsumowanie), **nie wywołuje** `writeFileSync`/`mkdirSync`. Prefiks odpowiedzi `**[dry-run]**`.

- [ ] **Step 1: Zinwentaryzuj punkty zapisu** — w każdym z 6 plików znajdź `writeFileSync`/`mkdirSync`/`writeFile` i miejsce, gdzie generowana jest treść (dla script_create już istnieje wzorzec: przy istniejącym pliku zwraca kod bez zapisu — uogólnij go). Zanotuj, które narzędzia piszą wiele plików (`mod` scaffold, `scenario_create_conflict` jeśli w zakresie — trzymaj się 6 z listy).
- [ ] **Step 2: Failing testy** `tests/tools/dry-run.test.ts` — dla każdego narzędzia: wywołaj z `dryRun:true` na katalogu tymczasowym, asercja: (a) odpowiedź zawiera `[dry-run]` i planowaną ścieżkę, (b) **plik NIE powstał** (`existsSync` === false). Uruchom → FAIL.
- [ ] **Step 3: Implementacja** — wspólny wzorzec: wydziel generowanie treści od zapisu; przy `dryRun` pomiń zapis i zwróć preview. Trzymaj DRY — jeśli powtarzalne, mały helper `formatDryRun(files: {path,content}[])` w `src/utils/`. PASS.
- [ ] **Step 4: Weryfikacja braku regresji** — normalny tryb (`dryRun:false`/pominięty) zapisuje jak dotąd; istniejące testy zielone. `npm run build && npm test`.
- [ ] **Step 5: Dokumentacja** — README tabela: dopisz `dryRun` przy narzędziach; `description` każdego narzędzia wspomina dry-run. Odhacz #10 w UPGRADE_IDEAS.md.
- [ ] **Step 6: Commit + PR** (`feat/dry-run-mutation-tools`).
